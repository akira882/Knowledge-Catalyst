# 画面・体験設計 (UX Design)

## 1. ターゲットユーザー定義

| ユーザー属性 | 特徴と課題 | コンセプト |
| :--- | :--- | :--- |
| **Beginner (新入社員)** | 「何を学ぶべきか」の地図を持っていない | **Guide - 迷わせない教育** |
| **Advanced (活用層)** | 特定の課題解決のために「ピンポイント」で知りたい | **Support - 思考の加速** |

## 2. インターフェース設計

### 2.1 新入社員用「クエスト画面」
*   **構造**: RPGのようなロードマップ形式。
*   **ワイヤーフレームイメージ**:
    *   [ ] Stage 1: 生成AIの倫理と基礎（必修）
    *   [ ] Stage 2: 広告コピー生成実践
    *   [ ] Stage 3: データ分析自動化
*   **仕掛け**: 進捗率に応じたランキング。バッジによる習熟度の証明。

### 2.2 活用層用「Copilot画面」
*   **挙動**: SlackやChrome拡張機能からの呼び出しを想定。
*   **機能**: 
    *   「今やっている業務」をテキスト入力すると、過去の全学習コンテンツから最適な「技」を提案。
    *   「昨日の社内勉強会で言っていたあの手法」という曖昧な検索への対応。

## 3. 評価フィードバックループ（ROI報告ボタン）
コンテンツ消費直後に以下のUIを表示します。

> **「このコンテンツで、あなたの業務は何分削減できそうですか？」**
> [ 5分 / 15分 / 30分以上 ]
> [ プロンプトをコピーして保存 ]

このデータが蓄積されることで、以下の「数値による成果実証」が可能になります。
*   **組織別ランキング**: どの本部が最もAIで時間を創出したか。
*   **コンテンツ自動選別**: ROIの低いコンテンツを管理者にアラート表示、または自動非表示化。
